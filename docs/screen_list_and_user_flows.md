## 2. User Flows

### Flow 1: 최초 로그인 → 프로필 작성

신규 사용자가 Google 로그인 후 프로필을 완성하는 흐름.

```
L-01 랜딩 페이지
 │  [로그인] 클릭
 ▼
A-01 로그인
 │  [Google로 로그인] 클릭
 │  → Google 인증 → OAuth 콜백
 │  ├─ @snu.ac.kr 아님: 에러 메시지 표시, 로그아웃
 │  └─ @snu.ac.kr 맞음 (최초 로그인) ▼
P-01 프로필 작성
 │  학과, 학년, 자기소개, 기술스택 입력
 │  [저장] 클릭
 ▼
L-01 랜딩 페이지 (로그인 상태)
```

**관련 스토리:** US-1.1, US-2.1

---

### Flow 2: 재로그인

기존 사용자가 로그인하는 흐름.

```
L-01 랜딩 페이지
 │  [로그인] 클릭
 ▼
A-01 로그인
 │  [Google로 로그인] 클릭
 │  → Google 인증 → OAuth 콜백
 │  ├─ 실패: 에러 메시지 표시
 │  └─ 성공 (프로필 이미 있음) ▼
L-01 랜딩 페이지 (로그인 상태)
```

**관련 스토리:** US-1.1

---

### Flow 3: 모집글 작성 (팀장)

팀장이 공모전 팀원 모집글을 작성하는 흐름.

```
G-01 헤더
 │  [모집글 작성] 클릭
 │  ├─ 비로그인: A-01 로그인으로 이동
 │  └─ 로그인 상태 ▼
R-03 모집글 작성
 │  공모전명, 마감일, 모집인원, 역할, 팀소개 입력
 │  [등록하기] 클릭
 │  ├─ 실패: 유효성 에러 표시
 │  └─ 성공 ▼
R-02 모집글 상세 (방금 작성한 글)
```

**관련 스토리:** US-3.1

---

### Flow 4: 모집글 탐색 → 지원 (지원자)

지원자가 팀을 찾아 지원하는 핵심 흐름.

```
R-01 모집글 리스트
 │  검색 / 역할 필터 / 모집중 토글로 탐색
 │  [모집글 카드] 클릭
 ▼
R-02 모집글 상세
 │  공모전 정보, 팀 소개, 팀장 프로필 확인
 │  [지원하기] 클릭
 │  ├─ 비로그인: A-01 로그인으로 이동
 │  ├─ 이미 지원함: "지원완료" 비활성화 상태
 │  ├─ 모집완료: 버튼 비활성화
 │  └─ 가능 ▼
G-02 지원 모달
 │  내 프로필 요약 미리보기 확인
 │  지원 메시지 작성
 │  [제출] 클릭
 │  ├─ 실패: 에러 메시지
 │  └─ 성공 ▼
R-02 모집글 상세 (지원완료 상태로 변경)
 │  "지원이 완료되었습니다" 토스트 표시
```

**관련 스토리:** US-4.1, US-4.2, US-4.3, US-4.4, US-4.5

---

### Flow 5: 지원자 관리 (팀장)

팀장이 지원자를 검토하고 승인/거절하는 흐름.

```
경로 A: 알림을 통한 진입
  N-01 알림 목록 (또는 헤더 알림 벨)
   │  "OOO님이 지원했습니다" 알림 클릭
   ▼
  R-05 지원자 관리

경로 B: 마이페이지를 통한 진입
  M-01 마이페이지
   │  [내 모집글] 탭 → 모집글 클릭
   ▼
  R-02 모집글 상세
   │  [지원자 관리] 클릭 (팀장에게만 표시)
   ▼
  R-05 지원자 관리

──────────────────────

R-05 지원자 관리
 │  지원자 카드 목록 확인 (프로필 + 지원 메시지)
 │
 ├─ [승인] 클릭
 │   → 상태 ACCEPTED로 변경
 │   → 지원자에게 승인 알림 발송
 │   → 지원자 연락처(이메일) 표시
 │
 └─ [거절] 클릭
     → 상태 REJECTED로 변경
     → 지원자에게 거절 알림 발송
```

**관련 스토리:** US-5.1, US-5.2, US-5.3, US-6.1

---

### Flow 6: 모집 상태 변경 (팀장)

팀장이 모집을 마감하는 흐름.

```
R-02 모집글 상세 (팀장 본인 글)
 │  [모집완료로 변경] 클릭
 ▼
R-02 모집글 상세
 │  상태 뱃지 "모집완료"로 변경
 │  지원하기 버튼 비활성화
 │
 │  (필요 시) [다시 모집중으로 변경] 클릭
 ▼
R-02 모집글 상세
    상태 뱃지 "모집중"으로 복원
```

**관련 스토리:** US-3.3

---

### Flow 7: 지원 결과 확인 (지원자)

지원자가 본인의 지원 결과를 확인하는 흐름.

```
경로 A: 알림을 통한 확인
  G-01 헤더 → 알림 벨 (뱃지 표시)
   │  클릭
   ▼
  N-01 알림 목록
   │  "승인되었습니다" 또는 "거절되었습니다" 알림 확인
   │  [알림] 클릭
   ▼
  R-02 모집글 상세

경로 B: 마이페이지를 통한 확인
  M-01 마이페이지
   │  [내 지원현황] 탭
   │  각 지원 상태 뱃지 확인 (대기중 / 승인 / 거절)
   │  [모집글] 클릭
   ▼
  R-02 모집글 상세
```

**관련 스토리:** US-6.1, US-6.2, US-7.2

---

### Flow 8: 프로필 수정

사용자가 기존 프로필을 수정하는 흐름.

```
M-01 마이페이지
 │  [프로필 수정] 클릭
 ▼
P-01 프로필 수정 (기존 값 프리필)
 │  원하는 필드 수정
 │  [저장] 클릭
 ▼
M-01 마이페이지
    "저장되었습니다" 토스트 표시
```

**관련 스토리:** US-2.2
