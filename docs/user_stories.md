# User Stories (MVP)

## 1. 인증 (Auth)

### US-1.1 로그인

- **As a** 서울대 학생
- **I want to** 서울대 Google 계정(@snu.ac.kr)으로 로그인하고 싶다
- **So that** 서비스를 이용하고, 서울대 재학생임을 인증할 수 있다

**Acceptance Criteria:**

- [ ] "Google로 로그인" 버튼을 클릭하여 로그인할 수 있다
- [ ] @snu.ac.kr 이메일의 Google 계정만 허용된다
- [ ] @snu.ac.kr이 아닌 계정으로 시도 시 에러 메시지가 표시된다
- [ ] 최초 로그인 시 프로필 작성 페이지(/profile/edit)로 이동한다
- [ ] 이후 로그인 시 메인 페이지(/)로 이동한다
- [ ] 로그인 상태가 유지된다 (새로고침해도 로그아웃되지 않음)

### US-1.3 로그아웃

- **As a** 로그인한 사용자
- **I want to** 로그아웃하고 싶다
- **So that** 공용 기기에서 내 계정을 안전하게 보호할 수 있다

**Acceptance Criteria:**

- [ ] 헤더의 로그아웃 버튼을 클릭하면 로그아웃된다
- [ ] 로그아웃 후 로그인 페이지로 이동한다

---

## 2. 프로필

### US-2.1 프로필 작성

- **As a** 신규 가입 사용자
- **I want to** 내 학과, 학년, 자기소개, 기술 스택 등 프로필을 작성하고 싶다
- **So that** 팀에 지원할 때 팀장이 내 역량을 확인할 수 있다

**Acceptance Criteria:**

- [ ] 이름, 학과, 학년, 자기소개(200자 이내), 기술 스택(태그)을 입력할 수 있다
- [ ] 포트폴리오 URL은 선택적으로 입력할 수 있다
- [ ] 기술 스택은 미리 정의된 태그 목록에서 다중 선택할 수 있다
- [ ] 필수 필드를 비워두면 저장할 수 없다
- [ ] 저장 성공 시 확인 메시지가 표시된다

### US-2.2 프로필 수정

- **As a** 기존 사용자
- **I want to** 내 프로필 정보를 수정하고 싶다
- **So that** 변경된 역량이나 정보를 반영할 수 있다

**Acceptance Criteria:**

- [ ] /profile/edit 페이지에서 기존 정보가 미리 채워져 있다
- [ ] 모든 필드를 수정하고 저장할 수 있다
- [ ] 마이페이지에서 프로필 수정 페이지로 이동할 수 있다

---

## 3. 공모전 팀 모집글

### US-3.1 모집글 작성

- **As a** 팀장
- **I want to** 공모전 정보와 함께 팀원 모집글을 작성하고 싶다
- **So that** 필요한 역할의 팀원을 모집할 수 있다

**Acceptance Criteria:**

- [ ] 공모전 이름, 모집 마감일, 모집 인원, 모집 역할, 팀 소개를 입력할 수 있다
- [ ] 공모전 링크, 공모전 마감일은 선택적으로 입력할 수 있다
- [ ] 모집 역할은 태그 형태로 복수 선택할 수 있다
- [ ] 필수 필드를 비워두면 제출할 수 없다
- [ ] 작성 완료 후 해당 모집글 상세 페이지(/posts/[id])로 이동한다
- [ ] 모집글의 초기 상태는 "모집중"이다

### US-3.2 모집글 수정

- **As a** 팀장
- **I want to** 내가 작성한 모집글을 수정하고 싶다
- **So that** 변경된 모집 조건을 반영할 수 있다

**Acceptance Criteria:**

- [ ] 모집글 상세 페이지에서 수정 버튼이 작성자에게만 표시된다
- [ ] 기존 내용이 미리 채워진 수정 폼이 표시된다
- [ ] 수정 완료 후 상세 페이지로 돌아간다

### US-3.3 모집 상태 변경

- **As a** 팀장
- **I want to** 모집 상태를 "모집완료"로 변경하고 싶다
- **So that** 더 이상 지원을 받지 않을 수 있다

**Acceptance Criteria:**

- [ ] 팀장이 수동으로 모집 상태를 "모집완료"로 변경할 수 있다
- [ ] "모집완료" 상태의 글에는 지원 버튼이 비활성화된다
- [ ] 모집완료 → 모집중으로 다시 변경할 수 있다

---

## 4. 모집글 탐색 및 지원

### US-4.1 모집글 목록 조회

- **As a** 지원자
- **I want to** 현재 모집중인 팀 목록을 보고 싶다
- **So that** 관심 있는 공모전 팀을 찾을 수 있다

**Acceptance Criteria:**

- [ ] /posts 페이지에 모집글이 최신순으로 표시된다
- [ ] 각 카드에 공모전 이름, 모집 역할, 모집 마감일, 현재 지원자 수가 표시된다
- [ ] 모집중/모집완료 상태가 뱃지로 구분된다

### US-4.2 모집글 필터링

- **As a** 지원자
- **I want to** 역할 태그나 모집 상태로 모집글을 필터링하고 싶다
- **So that** 내 역량에 맞는 팀을 빠르게 찾을 수 있다

**Acceptance Criteria:**

- [ ] 역할 태그로 필터링할 수 있다 (예: "프론트엔드"만 보기)
- [ ] "모집중만 보기" 토글로 모집완료 글을 숨길 수 있다
- [ ] 필터 초기화 버튼이 있다

### US-4.3 모집글 검색

- **As a** 지원자
- **I want to** 공모전 이름으로 모집글을 검색하고 싶다
- **So that** 특정 공모전의 팀을 찾을 수 있다

**Acceptance Criteria:**

- [ ] 검색창에 키워드를 입력하면 공모전 이름에서 검색된다
- [ ] 검색 결과가 없을 때 안내 메시지가 표시된다

### US-4.4 모집글 상세 조회

- **As a** 지원자
- **I want to** 모집글의 상세 정보를 확인하고 싶다
- **So that** 지원 여부를 결정할 수 있다

**Acceptance Criteria:**

- [ ] 공모전 정보, 팀 소개, 모집 역할, 마감일 등 모든 정보가 표시된다
- [ ] 팀장의 프로필(이름, 학과, 기술 스택)이 요약 표시된다
- [ ] "지원하기" 버튼이 표시된다 (비로그인 시 로그인 페이지로 이동)

### US-4.5 팀 지원

- **As a** 지원자
- **I want to** 관심 있는 팀에 지원 메시지와 함께 지원하고 싶다
- **So that** 팀장이 내 프로필과 지원 동기를 확인할 수 있다

**Acceptance Criteria:**

- [ ] "지원하기" 버튼 클릭 시 지원 메시지 입력 모달이 표시된다
- [ ] 지원 메시지를 작성하고 제출할 수 있다
- [ ] 제출 시 내 프로필 정보가 자동으로 함께 전달된다
- [ ] 같은 모집글에 중복 지원할 수 없다 (이미 지원한 경우 "지원완료" 표시)
- [ ] 지원 완료 시 확인 메시지가 표시된다
- [ ] 모집완료 상태의 글에는 지원할 수 없다

---

## 5. 지원자 관리

### US-5.1 지원자 목록 확인

- **As a** 팀장
- **I want to** 내 모집글에 지원한 사람들의 목록을 확인하고 싶다
- **So that** 적합한 팀원을 선별할 수 있다

**Acceptance Criteria:**

- [ ] /posts/[id]/applicants 페이지에 지원자 목록이 표시된다
- [ ] 각 지원자의 프로필(이름, 학과, 학년, 기술 스택, 포트폴리오)이 표시된다
- [ ] 지원 메시지가 함께 표시된다
- [ ] 지원 상태(대기중/승인/거절)가 표시된다
- [ ] 작성자가 아닌 사용자는 이 페이지에 접근할 수 없다

### US-5.2 지원 승인

- **As a** 팀장
- **I want to** 적합한 지원자를 승인하고 싶다
- **So that** 팀원으로 확정할 수 있다

**Acceptance Criteria:**

- [ ] "승인" 버튼 클릭 시 해당 지원자 상태가 ACCEPTED로 변경된다
- [ ] 승인된 지원자에게 알림이 발송된다
- [ ] 승인된 지원자의 연락처(이메일)가 팀장에게 표시된다

### US-5.3 지원 거절

- **As a** 팀장
- **I want to** 부적합한 지원자를 거절하고 싶다
- **So that** 다른 지원자를 검토하는 데 집중할 수 있다

**Acceptance Criteria:**

- [ ] "거절" 버튼 클릭 시 해당 지원자 상태가 REJECTED로 변경된다
- [ ] 거절된 지원자에게 알림이 발송된다

---

## 6. 알림

### US-6.1 알림 수신

- **As a** 사용자
- **I want to** 내 지원/모집 활동에 대한 알림을 받고 싶다
- **So that** 중요한 변경 사항을 놓치지 않을 수 있다

**Acceptance Criteria:**

- [ ] 팀장: 새로운 지원이 들어오면 알림을 받는다
- [ ] 지원자: 지원이 승인되면 알림을 받는다
- [ ] 지원자: 지원이 거절되면 알림을 받는다
- [ ] 헤더에 읽지 않은 알림 개수가 뱃지로 표시된다

### US-6.2 알림 목록 확인

- **As a** 사용자
- **I want to** 알림 목록을 확인하고 싶다
- **So that** 과거 알림 내역을 조회할 수 있다

**Acceptance Criteria:**

- [ ] /notifications 페이지에 알림이 최신순으로 표시된다
- [ ] 각 알림에 발생 시간이 표시된다
- [ ] 읽지 않은 알림과 읽은 알림이 시각적으로 구분된다
- [ ] 알림 클릭 시 해당 모집글로 이동하며 읽음 처리된다

---

## 7. 마이페이지

### US-7.1 내 모집글 관리

- **As a** 팀장
- **I want to** 내가 작성한 모집글 목록을 한눈에 보고 싶다
- **So that** 각 모집글의 지원 현황을 빠르게 파악할 수 있다

**Acceptance Criteria:**

- [ ] 마이페이지에 내가 작성한 모집글 목록이 표시된다
- [ ] 각 모집글의 상태(모집중/완료)와 지원자 수가 표시된다
- [ ] 모집글 클릭 시 해당 상세 페이지로 이동한다

### US-7.2 내 지원 현황 확인

- **As a** 지원자
- **I want to** 내가 지원한 팀들의 현황을 확인하고 싶다
- **So that** 승인/거절 결과를 한눈에 볼 수 있다

**Acceptance Criteria:**

- [ ] 마이페이지에 내가 지원한 팀 목록이 표시된다
- [ ] 각 지원의 상태(대기중/승인/거절)가 뱃지로 표시된다
- [ ] 지원한 모집글 클릭 시 해당 상세 페이지로 이동한다

---

## 8. 랜딩 페이지

### US-8.1 서비스 소개

- **As a** 신규 방문자
- **I want to** 이 서비스가 무엇인지 빠르게 이해하고 싶다
- **So that** 가입할지 결정할 수 있다

**Acceptance Criteria:**

- [ ] 서비스 소개 문구가 표시된다
- [ ] 최근 모집글 일부가 미리보기로 표시된다
- [ ] 회원가입/로그인 CTA 버튼이 표시된다
- [ ] 비로그인 사용자에게만 CTA가 표시되고, 로그인 사용자는 바로 모집글 탐색 가능
