# Product Requirements Document (MVP)

## 1. 제품 개요

**제품명:** SNU TeamUp (가칭)

**한줄 요약:** 서울대학교 학생들이 공모전에 참여할 때, 적합한 팀원을 효율적으로 찾을 수 있도록 돕는 공모전 팀빌딩 웹사이트.

**핵심 가치:**

- 공모전별 팀 모집 가능: 참가하고 싶은 공모전을 직접 올리고 팀원을 모집할 수 있음
- 자기소개/경력/포트폴리오 기반 지원: 지원자의 경험과 역량을 한눈에 확인할 수 있어 더 적합한 팀 구성이 가능함
- 방장 승인형 팀 구성: 팀장이 지원자를 검토하고 승인하는 구조로 신뢰도 높은 팀빌딩이 가능함

**대상 사용자:** 서울대학교 재학생 (학부생 + 대학원생)

---

## 2. 사용자 유형

| 유형 | 설명 |
| --- | --- |
| **팀장 (Host)** | 공모전 팀 모집글을 작성하고, 지원자를 검토/승인하는 사용자 |
| **지원자 (Applicant)** | 모집글을 탐색하고, 관심 있는 팀에 지원하는 사용자 |

> 한 사용자가 팀장과 지원자 역할을 동시에 수행할 수 있음

---

## 3. 핵심 기능 (MVP Scope)

### 3.1 인증 (Auth)

- Google 로그인 (Supabase Auth) — 서울대 메일(@snu.ac.kr) 계정만 허용
- 로그인 상태 유지 (세션/JWT)

### 3.2 프로필

사용자가 한 번 작성하면, 지원 시 자동으로 팀장에게 전달됨.

| 필드 | 필수 여부 | 설명 |
| --- | --- | --- |
| 이름 | 필수 | 실명 |
| 학과 | 필수 | 전공/부전공 |
| 학년 | 필수 | 학부 1~4학년 / 석사 / 박사 |
| 자기소개 | 필수 | 간단한 소개 (200자 이내) |
| 기술 스택/역량 | 필수 | 태그 형태로 선택 (예: React, Python, 디자인, 기획, 마케팅 등) |
| 포트폴리오 URL | 선택 | GitHub, Behance, 개인 사이트 등 링크 |
| 연락처 | 필수 | 이메일 (기본값: 가입 이메일) |

### 3.3 공모전 팀 모집글 작성 (팀장)

팀장이 모집글을 생성하여 팀원을 모집.

| 필드 | 필수 여부 | 설명 |
| --- | --- | --- |
| 공모전 이름 | 필수 | 참여할 공모전명 |
| 공모전 링크 | 선택 | 공모전 상세 페이지 URL |
| 공모전 마감일 | 선택 | 공모전 제출 마감일 |
| 모집 마감일 | 필수 | 팀원 모집 마감일 |
| 모집 인원 | 필수 | 필요한 팀원 수 (예: 2명) |
| 모집 역할 | 필수 | 필요한 역할 태그 (예: 프론트엔드, 디자이너) |
| 팀 소개 | 필수 | 프로젝트 방향, 진행 방식 등 설명 |
| 모집 상태 | 자동 | 모집중 / 모집완료 |

### 3.4 모집글 탐색 및 지원 (지원자)

- **모집글 리스트:** 최신순으로 모집중인 글 목록 표시
- **필터링:** 역할 태그, 모집 상태(모집중만 보기)로 필터
- **검색:** 공모전 이름 키워드 검색
- **지원하기:** 버튼 클릭 → 간단한 지원 메시지 작성 후 제출
  - 지원 시 프로필 정보가 자동으로 함께 전달됨
  - 한 모집글에 중복 지원 불가

### 3.5 지원자 관리 (팀장)

- **지원자 목록 확인:** 모집글별 지원자 리스트 (프로필 + 지원 메시지)
- **승인/거절:** 팀장이 각 지원자를 승인 또는 거절
- **승인 시:** 지원자에게 승인 알림 (사이트 내 알림)
- **모집 완료:** 인원이 다 차거나 수동으로 모집 상태를 "모집완료"로 변경

### 3.6 알림

MVP에서는 사이트 내 알림만 지원 (이메일 알림은 추후).

| 알림 트리거 | 수신자 | 내용 |
| --- | --- | --- |
| 새로운 지원 | 팀장 | "OOO님이 [공모전명] 팀에 지원했습니다" |
| 지원 승인 | 지원자 | "[공모전명] 팀에 승인되었습니다" |
| 지원 거절 | 지원자 | "[공모전명] 팀 지원이 거절되었습니다" |

### 3.7 마이페이지

- **내 프로필 수정**
- **내가 만든 모집글** 목록 (팀장 시점)
- **내가 지원한 팀** 목록 + 상태 (대기중/승인/거절)

## 4. 페이지 구조

```
/ .......................... 랜딩 페이지 (서비스 소개 + 모집글 미리보기)
/login .................... 로그인 (Google OAuth)
/posts .................... 모집글 리스트 (탐색 + 검색 + 필터)
/posts/new ................ 모집글 작성
/posts/[id] ............... 모집글 상세 (지원하기 버튼 포함)
/posts/[id]/applicants .... 지원자 관리 (팀장 전용)
/profile/edit ............. 프로필 수정
/mypage ................... 마이페이지 (내 모집글 + 내 지원 현황)
/notifications ............ 알림 목록
```

## 5. 데이터 모델

### User

```
id
email (snu.ac.kr, Google OAuth)
name
department (학과)
grade (학년)
bio (자기소개)
skills [] (기술/역량 태그)
portfolioUrl?
createdAt
```

### Post (모집글)

```
id
authorId → User
contestName (공모전명)
contestUrl?
contestDeadline?
recruitDeadline (모집 마감일)
recruitCount (모집 인원)
roles [] (모집 역할 태그)
description (팀 소개)
status: RECRUITING | CLOSED
createdAt
```

### Application (지원)

```
id
postId → Post
applicantId → User
message (지원 메시지)
status: PENDING | ACCEPTED | REJECTED
createdAt
```

### Notification (알림)

```
id
userId → User
type: NEW_APPLICATION | ACCEPTED | REJECTED
message
postId → Post
isRead: boolean
createdAt
```

## 7. MVP에서 제외하는 기능 (Post-MVP)

| 기능 | 이유 |
| --- | --- |
| 실시간 채팅 | 복잡도 높음. MVP에서는 승인 후 연락처 교환으로 대체 |
| 이메일 알림 | 사이트 내 알림으로 우선 대체 |
| 공모전 DB 자동 크롤링 | 사용자가 직접 공모전 정보를 입력하는 것으로 충분 |
| 팀 활동 관리 (일정, 칸반) | 팀 매칭까지가 MVP 범위. 이후 활동은 외부 도구 사용 |
| 리뷰/평점 시스템 | 사용자 풀이 충분해진 후 도입 |
| 추천 알고리즘 | 초기에는 수동 탐색으로 충분 |
| 관리자 페이지 | 초기에는 DB 직접 관리로 대체 |
